## 1. Контекст

На основе предварительного Технического проекта  предлагается поэтапная разработка системы управленческой отчётности для группы компаний «Открытие» (3 юридических лица, 3 ИП).

**Цель:** Построить работающий ETL-процесс «1С → Битрикс24» и последовательно реализовать 7 финансовых отчётов первой очереди.

---

## 2. Принцип оплаты

**Оплата — по факту потраченных часов, 5 000 руб./час.**

- В конце каждого этапа подбивается фактическое количество часов
- Выставляется акт и счёт на оплату
- Предварительная оценка часов по каждому этапу указана ниже (финальная сумма может незначительно отклоняться)
- Заказчик утверждает этап → Исполнитель приступает → по завершению — акт и оплата

Такой подход позволяет:
- Гибко реагировать на уточнения требований в процессе
- Не переплачивать за «жёсткую смету» с запасом
- Платить только за реально выполненную работу

## 3. Этапы работ

### Этап 0 — Архитектура и пилотный ETL (≈ 20–60 часов)

**Цель:** Построить и проверить сквозной ETL-процесс на одном отчёте.

**Состав работ:**
1. Настройка соединения Python ↔ 1С (COM / выгрузка в CSV)
2. Разработка ETL-скрипта: выгрузка данных из 1С → трансформация → загрузка в Битрикс24
3. Реализация **одного пилотного отчёта** (например, ОПиУ) «под ключ»
4. Проверка корректности данных, отладка
5. Выпуск **методической документации** по пилотному отчёту:
   - Какие документы/регистры 1С используются
   - Какая бизнес-логика трансформации применяется
   - Какие поля и формулы используются
   - Как данные загружаются в Битрикс24

**Результат этапа:**
- Работающий ETL-пайплайн
- Один готовый отчёт в Битрикс24
- Методика построения отчёта (документ)
- Понимание реальной трудоёмкости для следующих этапов

**Ориентировочная стоимость:** 20–60 ч × 5 000 = **100 000 – 300 000 руб.**

## 5. Методическая документация

Каждый этап сопровождается выпуском методической документации — это отдельный документ, который описывает **бизнес-методику построения отчёта**:

- **Источники данных:** какие документы, регистры, справочники 1С используются
- **Бизнес-логика:** какие данные выбираются, по каким правилам, какие фильтры применяются
- **Формулы и расчёты:** как считаются показатели (валовая маржа, EBITDA, CCC, DSO и т.д.)
- **Трансформация:** как данные преобразуются перед загрузкой в Битрикс24
- **Визуализация:** как отчёт выглядит в Битрикс24, какие разрезы доступны

Эта документация остаётся у заказчика и может использоваться для:
- Проверки корректности расчётов
- Обучения новых сотрудников
- Доработки отчётов силами заказчика в будущем

## 6. Порядок взаимодействия

1. Заказчик утверждает этап (письменно или в чате)
2. Исполнитель приступает к работам
3. В процессе — оперативная обратная связь по уточнениям
4. По завершению этапа:
   - Исполнитель демонстрирует результат
   - Передаёт методическую документацию
   - Подбивает фактические часы
   - Выставляет акт и счёт
5. Заказчик оплачивает этап
6. Переход к следующему этапу
