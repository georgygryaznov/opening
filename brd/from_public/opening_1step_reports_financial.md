## 1. Общие положения

### 1.1. Цель документа
Данный документ — техническое задание на разработку первой очереди системы управленческой отчётности для ГК «Открытие». Первая очередь включает **финансовые и экономические отчёты**.

### 1.2. Бизнес-контекст
- **Группа компаний:** Открытие, Открытие Тюмень, Открытие Пермь, ИП Зиновьев, ИП Семенов, ИП Ушакова (всего 6 юрлиц)
- **Направления:** оптово-розничная торговля стройматериалами, грузоперевозки
- **Штат:** ~50 человек
- **Бухгалтерия:** одна на все компании
- **Валюта отчётности:** российский рубль (RUB)

### 1.3. Принципы построения отчётности
1. **Множественность разрезов** — каждый отчёт строится по каждому юрлицу отдельно + консолидированно по ГК
2. **Исключение внутригрупповых оборотов** — при консолидации выручка, расходы и обязательства между юрлицами ГК исключаются
3. **Сравнительный анализ** — месяц к месяцу, квартал к кварталу, год к году, процент изменений
4. **Цветовая индикация** (при наличии технической возможности) — зелёный (норма), жёлтый (риск), красный (проблема)
5. **Drill-down** (при наличии технической возможности) — клик на цифру → проваливание в детализацию (регион → клиент → заказ)

## 2. Архитектура решения

### 2.1. Общая схема

> **Примечание:** Архитектура решения, состав компонентов и схема взаимодействия будут уточняться по ходу реализации первой очереди.

```
┌─────────────────────┐     ┌──────────────────┐     ┌─────────────────────┐     ┌─────────────────────┐
│                     │     │                  │     │                     │     │                     │
│   1С УТ 11.6        │────>│  ETL-процесс     │────>│  DWH / витрина      │────>│  Битрикс24          │
│   (источник данных) │     │  (выгрузка +     │     │  управленческих     │     │  (площадка          │
│                     │     │   трансформация) │     │  данных             │     │   отчётов)          │
└─────────────────────┘     └──────────────────┘     └─────────────────────┘     └─────────────────────┘
```

### 2.2. Компоненты

| Компонент | Технология | Назначение |
|---|---|---|
| Источник данных | 1С УТ 11.6 | Хранение первичных данных (продажи, закупки, ДС, остатки) |
| ETL-слой | Python-скрипты / 1С-обработки | Выгрузка, очистка, трансформация, агрегация данных |
| DWH / витрина данных | PostgreSQL / MS SQL / иная согласованная БД | Промежуточное хранение очищенных, нормализованных и агрегированных данных; исключение внутригрупповых оборотов; контрольные сверки с 1С |
| Площадка отчётов | Битрикс24 (API 61) | Визуализация, дашборды, доступ для пользователей |

### 2.3. Варианты реализации ETL

> **Вариант A (рекомендуемый): Внешний ETL (Python)**
> Python-скрипты подключаются к 1С через COM-соединение или выгружают данные через обработки 1С в CSV/JSON. Скрипты выполняют трансформацию и загружают данные в Битрикс24 через REST API.
> Конкретный механизм интеграции с 1С будет выбран после полной технической проверки доступа к базе 1С, инфраструктуры заказчика и требований безопасности. COM-соединение возможно только при запуске ETL в Windows-среде.
> 
> **Плюсы:** гибкость, версионирование кода в Git, независимость от версий 1С
> **Минусы:** требуется настройка соединения с 1С

> **Вариант B: Внутренние обработки 1С**
> Все отчёты формируются непосредственно в 1С и публикуются в Битрикс24 через HTTP-запросы.
> 
> **Плюсы:** не требуется внешних компонентов
> **Минусы:** сложность версионирования, привязка к платформе 1С

## 3. Источник данных: 1С УТ 11.6

### 3.1. Общие сведения

> **Примечание:** Состав объектов метаданных 1С, регистров и документов будет уточняться по ходу реализации первой очереди.

- **Одна база 1С** на все 6 юрлиц
- Все юрлица ведутся в единой информационной базе
- Справочники (номенклатура, контрагенты, статьи движения ДС) — **единые** для всех юрлиц

### 3.2. Необходимые регистры и документы

Для построения финансовой отчётности потребуются следующие объекты метаданных 1С:

| Объект 1С | Назначение | Используется в отчётах |
|---|---|---|
| Документ «Реализация товаров и услуг» | Выручка, себестоимость | ОПиУ, План-факт |
| Документ «Поступление товаров и услуг» | Закупки, себестоимость | ОПиУ |
| Документ «Поступление безналичных ДС» | Приход денег | ОДДС |
| Документ «Списание безналичных ДС» | Расход денег | ОДДС |
| Документ «Приходный кассовый ордер» | Приход наличных | ОДДС |
| Документ «Расходный кассовый ордер» | Расход наличных | ОДДС |
| Регистр «Остатки товаров» | Складские остатки | Упр. баланс |
| Регистр «Взаиморасчёты с контрагентами» | ДЗ и КЗ | Дебиторка, Упр. баланс |
| Регистр «Денежные средства» | Остатки на счетах | Упр. баланс, Dashboard |
| Управленческие данные по расходам, ОС, кредитам и займам / бухгалтерская база при необходимости | Расходы, ОС, обязательства | ОПиУ, Упр. баланс |

### 3.3. Требования к справочникам

> **Критично:** Перед запуском отчётности необходимо проверить и привести к единому виду:
> - Статьи движения денежных средств
> - Номенклатурные группы
> - Статьи доходов и расходов

### 4.2. Механизмы реализации

Для размещения отчётов в Битрикс24 могут использоваться следующие возможности:

| Механизм | Назначение |
|---|---|
| **Виджеты на портале** | Размещение дашбордов на рабочих столах сотрудников |
| **Highload-блоки / инфоблоки / списки / смарт-процессы / кастомный модуль** | Возможные варианты хранения и отображения агрегированных данных |
| **REST API (batch)** | Массовая загрузка данных из ETL |
| **Бизнес-процессы** | Автоматическое уведомление об отклонениях |
| **Отчёты / виджеты / дашборды** | Построение табличных отчётов и визуализаций |

### 4.3. Структура хранения данных в Битрикс24

Для каждого отчёта создаётся отдельная сущность / витрина со своей структурой полей. Конкретный тип сущности определяется после технической проверки Битрикс24. Данные загружаются агрегированными (не первичные документы, а уже свёрнутые по разрезам).

## 5. Отчёт 1.1 — ОПиУ (P&L)

### 5.1. Назначение
Отчёт о прибылях и убытках — показывает реальную доходность бизнеса за период.

> **Примечание:** Состав показателей, формулы расчёта и источники данных будут уточняться по ходу реализации первой очереди.

### 5.2. Разрезы
- Юрлицо (6 шт. + консолидация)
- Период (месяц, квартал, год)
- Сравнение: тек. месяц / прошлый месяц / аналог. период прошлого года / % изменений

### 5.3. Структура данных (поля таблицы)

| Поле | Русское название | Тип | Источник | Формула / Примечание |
|---|---|---|---|---|
| `period` | Период | Дата (месяц) | — | Период отчёта |
| `legal_entity_id` | Юрлицо | Строка | Справочник «Организации» | Юрлицо |
| `revenue` | Выручка | Число | Док. «Реализация товаров и услуг» | Сумма реализации за вычетом внутренних продаж |
| `cogs` | Себестоимость | Число | Док. «Реализация» (себестоимость) | Себестоимость проданных товаров |
| `gross_profit` | Валовая прибыль | Число | Расчёт | `revenue - cogs` |
| `gross_margin_pct` | Валовая маржа, % | Число | Расчёт | `gross_profit / revenue * 100` |
| `commercial_expenses` | Коммерческие расходы | Число | Управленческие статьи расходов / источник требует проверки | Коммерческие расходы |
| `admin_expenses` | Управленческие расходы | Число | Управленческие статьи расходов / источник требует проверки | Управленческие расходы |
| `ebitda` | EBITDA | Число | Расчёт | `gross_profit - commercial_expenses - admin_expenses` |
| `net_profit` | Чистая прибыль | Число | Расчёт | `ebitda - проценты - налоги` |
| `contribution_margin` | Маржинальный доход | Число | Расчёт | `revenue - cogs - logistics - marketing` |
| `is_consolidated` | Консолидировано | Булево | — | Признак консолидированной строки |

### 5.4. Особенности расчёта
- **Внутригрупповые обороты** исключаются на этапе ETL: если организация А продала организации Б — эта сумма вычитается из выручки при консолидации
- **Contribution Margin** — маржа после переменных затрат (логистика + маркетинг вычитаются дополнительно)

> **Примечание по источникам расходов:** если в 1С УТ 11.6 нет полноценного бухгалтерского контура, расходы строятся по управленческим статьям, документам и регистрам УТ. Если нужна бухгалтерская точность по счетам, подключается дополнительный источник — 1С:Бухгалтерия / бухгалтерская база.

### 5.5. Визуализация в Битрикс24
- Таблица с группировкой по юрлицам
- Строка «Итого по ГК» (консолидированная)
- Цветовая индикация: Gross Margin < 15% — красный, 15–25% — жёлтый, > 25% — зелёный
- Drill-down: клик на выручку → провал в регионы → клиенты → заказы

## 6. Отчёт 1.2 — ОДДС (Cash Flow)

### 6.1. Назначение
Отчёт о движении денежных средств — реальные поступления и расходы, сравнение плана с фактом.

> **Примечание:** Состав показателей, формулы расчёта и источники данных будут уточняться по ходу реализации первой очереди.

### 6.2. Разрезы
- Юрлицо (6 шт. + консолидация)
- Период (месяц, квартал, год)
- План / Факт / Отклонение

### 6.3. Структура данных

| Поле | Русское название | Тип | Источник | Примечание |
|---|---|---|---|---|
| `period` | Период | Дата (месяц) | — | Период отчёта |
| `legal_entity_id` | Юрлицо | Строка | Справочник «Организации» | Юрлицо |
| `operational_inflow` | Операционный приток | Число | ПКО + Поступление б/н | Поступления от основной деятельности |
| `operational_outflow` | Операционный отток | Число | РКО + Списание б/н | Расходы на основную деятельность |
| `operational_net` | Чистый операционный поток | Число | Расчёт | `inflow - outflow` |
| `investment_inflow` | Инвестиционный приток | Число | Поступления от продажи ОС | Инвестиционный приток |
| `investment_outflow` | Инвестиционный отток | Число | Покупка ОС, кап. вложения | Инвестиционный отток |
| `investment_net` | Чистый инвестиционный поток | Число | Расчёт | `investment_inflow - investment_outflow` |
| `financial_inflow` | Финансовый приток | Число | Кредиты, займы, лизинг | Финансовый приток |
| `financial_outflow` | Финансовый отток | Число | Погашение кредитов, % | Финансовый отток |
| `financial_net` | Чистый финансовый поток | Число | Расчёт | `financial_inflow - financial_outflow` |
| `total_net_cashflow` | Чистый денежный поток | Число | Расчёт | `operational_net + investment_net + financial_net` |
| `plan_inflow` | Плановые поступления | Число | Бюджет ДДС | Плановые поступления |
| `plan_outflow` | Плановые расходы | Число | Бюджет ДДС | Плановые расходы |
| `deviation_pct` | Отклонение, % | Число | Расчёт | `(fact - plan) / plan * 100` |

### 6.4. Категории расходов (статьи ДДС)

Расходы детализируются по категориям:
- Закуп товара
- Аренда
- ФОТ
- Налоги
- Транспортные расходы
- Прочие операционные расходы

### 6.5. Особенности
- **Внутренние переброски денег** между юрлицами исключаются при консолидации
- Отклонение > 10% — сигнал для разбора (цветовая индикация)
- **Критическое предупреждение:** прибыль ≠ деньги. Компания может быть прибыльной по ОПиУ и иметь кассовый разрыв

### 6.6. Визуализация
- Таблица «План vs Факт» с цветовой индикацией отклонений
- График: притоки / оттоки / чистый поток по месяцам
- Drill-down: клик на категорию расходов → детализация по документам

## 7. Отчёт 1.3 — Управленческий баланс

### 7.1. Назначение
Срез состояния бизнеса на конкретную дату.

> **Примечание:** Состав показателей, формулы расчёта и источники данных будут уточняться по ходу реализации первой очереди.

### 7.2. Разрезы
- Юрлицо (6 шт. + консолидация)
- Дата (на конец месяца / квартала / года)

### 7.3. Структура данных

| Поле | Русское название | Тип | Источник | Примечание |
|---|---|---|---|---|
| `date` | Дата среза | Дата | — | Дата среза |
| `legal_entity_id` | Юрлицо | Строка | Справочник «Организации» | Юрлицо |
| `cash_balance` | Остаток ДС | Число | Регистр «Денежные средства» | Остатки на всех счетах и в кассе |
| `inventory_balance` | Товарные запасы | Число | Регистр «Остатки товаров» | Товарные запасы на складах |
| `accounts_receivable` | Дебиторская задолженность | Число | Регистр «Взаиморасчёты» | Дебиторская задолженность (нам должны) |
| `fixed_assets` | Основные средства | Число | ОС (автопарк, оборудование) / источник требует проверки | Основные средства |
| `total_assets` | Активы, всего | Число | Расчёт | Сумма активов |
| `accounts_payable` | Кредиторская задолженность | Число | Регистр «Взаиморасчёты» | Кредиторская задолженность (мы должны) |
| `loans` | Заёмные средства | Число | Кредиты и займы / источник требует проверки | Заёмные средства |
| `liabilities_total` | Обязательства, всего | Число | Расчёт | Сумма обязательств |
| `equity` | Собственный капитал | Число | Расчёт | `total_assets - liabilities_total` |
| `passive_total` | Пассивы, всего | Число | Расчёт | `liabilities_total + equity`, контроль: `total_assets = passive_total` |
| `ccc_days` | Цикл конверсии денег (CCC), дни | Число | Расчёт | Cash Conversion Cycle (дни) |

> **Примечание по источникам баланса:** баланс первой очереди является управленческим и строится по доступным данным УТ: ДС, товарные остатки, ДЗ/КЗ, взаиморасчёты. Основные средства, кредиты, займы и собственный капитал включаются только при наличии источника данных в УТ либо при подключении дополнительной бухгалтерской базы.

### 7.4. Cash Conversion Cycle (CCC)
Формула: `Оборачиваемость запасов (дни) + DSO – DPO`

- **Оборачиваемость запасов (дни):** `Средний запас / Себестоимость продаж × 30`
- **DSO (Days Sales Outstanding):** `Средняя ДЗ / Выручка × 30`
- **DPO (Days Payable Outstanding):** `Средняя КЗ / Себестоимость × 30`

### 7.5. Визуализация
- Таблица «Активы / Пассивы» по юрлицам
- График динамики CCC по месяцам
- Цветовая индикация: CCC > 60 дней — красный, 30–60 — жёлтый, < 30 — зелёный

## 8. Отчёт 1.4 — Дебиторская задолженность

## 13. Консолидация данных

### 13.1. Принцип консолидации

> **Примечание:** Правила консолидации, состав исключаемых оборотов и алгоритмы будут уточняться по ходу реализации первой очереди.

Каждый отчёт существует в двух вариантах:
1. **По юрлицам** — отдельные строки для каждого из 6 юрлиц
2. **Консолидированный** — сумма по всем юрлицам с исключением внутригрупповых оборотов

### 13.2. Исключение внутригрупповых оборотов

| Тип оборота | Что исключается | Где |
|---|---|---|
| Продажи между юрлицами ГК | Выручка и себестоимость | ОПиУ |
| Переброски денег между юрлицами | Приход и расход ДС | ОДДС |
| Внутригрупповая ДЗ и КЗ | Взаимные обязательства | Упр. баланс |

### 13.3. Идентификация внутригрупповых операций
- Все контрагенты, входящие в ГК, помечаются признаком «Внутригрупповой»
- При консолидации операции с такими контрагентами исключаются

## 14. Регламент загрузки и обновления

### 14.1. Периодичность

> **Примечание:** Регламент загрузки, периодичность и правила обработки ошибок будут уточняться по ходу реализации первой очереди.

| Отчёт | Периодичность загрузки |
|---|---|
| ОПиУ | Ежедневно (накопительно) |
| ОДДС | Ежедневно |
| Управленческий баланс | Ежедневно |
| Дебиторская задолженность | Ежедневно |
| Платёжный календарь | Еженедельно |
| План-фактный анализ | Ежемесячно (после закрытия периода) |
| Dashboard | Ежедневно |

### 14.2. Обработка ошибок
- При ошибке загрузки — повтор через 1 час (до 3 попыток)
- Если все попытки неудачны — уведомление администратору
- Логирование всех этапов в файл `etl_log_YYYY_MM_DD.log`

### 14.3. Подтверждение отчётов
- Каждый отчёт должен быть подтверждён ответственным представителем заказчика
- При наличии замечаний заказчик передаёт письменный перечень замечаний
- Замечания фиксируются, классифицируются и учитываются при доработке отчёта
